Vulkan SDK for Mac Release Notes

Vulkan SDK 1.1.85.0 Release Notes

Version 1.1.85.0 for Mac
October 4, 2018

This SDK supports Vulkan API revision 1.1.85 on macOS (MoltenVK subset) and provides partial Vulkan support though the use of the MoltenVK library which is a "translation" or "porting" library that maps most of the Vulkan functionality to the underlying graphics support on macOS and iOS platforms. Please see the MoltenVK Runtime User Guide on the MoltenVK GitHub for more information. The prior SDK supported Vulkan API revision 1.1.82 (MoltenVK subset).

Much of the documentation for this SDK is contained in the Getting Started Guide, found in the Documentation folder in this SDK and on the LunarXchange website.

Overview of new features in SDK 1.1.85.0

  • On September 19, 2018, NVIDIA released new Vulkan extensions to support their Turing architecture. These new extensions are supported and covered by Vulkan validation layers and the SPIR-V toolchain.
  • Since the release of SDK 1.1.82.1, the following extensions have been added to the Vulkan spec and may be supported by a Vulkan device. To query the extensions available to a given physical device, call vkEnumerateDeviceExtensionProperties() or contact the hardware vendor.
    VK_EXT_astc_decode_mode
    VK_EXT_inline_uniform_block
    VK_KHR_vulkan_memory_model
    VK_NV_compute_shader_derivatives
    VK_NV_corner_sampled_image
    VK_NV_fragment_shader_barycentric
    VK_NV_mesh_shader
    VK_NV_representative_fragment_test
    VK_NV_scissor_exclusive
    VK_NV_shader_image_footprint
    VK_NV_shading_rate_image
    VK_NVX_raytracing
  • VUID validation coverage documentation is now included in the LunarG Vulkan SDK. This documentation (validation_error_database.csv and an HTML version) has been updated to represent the validation layer VUID coverage status as of this 1.1.85.0 SDK. validation_error_database.csv can be found locally in the SDK Documentation directory once the SDK is installed on the system. A spreadsheet editor can then be used to sort and view the data. Additionally, an HTML version of the validation error database can be found here.
  • Bug fixes (see the SDK Components section below)
  • Improved validation coverage

Developer Notes

  • In the next Vulkan SDK release, cube and via will be renamed vkcube and vkvia respectively. SDK 1.1.85.0 will be the final Vulkan SDK to employ the names cube and via.
  • The standard Khronos validation layers have been migrated to use the canonical Vulkan VUID values, which are strings in the form of "VUID-vkCmdBlitImage-filter-00236". In doing so, the old integer-based enumerated values have been phased out and are no longer returned by log_msg() calls.
    • Developers keying on these returned values for their development should use the map in the vk_validation_error_messages.h file to obtain the matching Vulkan VUIDs when performing their updates. This file can be found in the layers directory of the Vulkan-ValidationLayers GitHub repository.
